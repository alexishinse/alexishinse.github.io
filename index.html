A propos

Typhon est une application Android qui permet de jouer des fichiers audio de type MP3. Cette application vous permet de jouer vos titres favoris dans une interface conviviale. Notre application permet de modifier la progression d'une chanson et elle contient la plupart des actions de base d'un lecteur de fichier audio.

Notre application est un projet opensource et n'est que le commencement d'une application grandissante, il vous est libre de contribuer au projet, afin que tout le monde puisse profiter d'une application qui assouvi le besoins des utilisateurs.

A propos de Typhon

Typhon est une application android qui permet de faire la lecture de fichier audio, programme par Alexis Hinse et Gabrielle Espatia.

Copyright © Typhon Corp 2016

Une application portative

Notre application effectue la lecture de fichier audio de type MP3. L'utilisateur peut modifier la progression d'une trame sonore et celui-ci est capable de mofifier le ton du volume. De plus, l'application permet d'afficher l'image de l'album correspondant a celle qui est en lecture.

Application qui occupe peu d'espace

Notre application demande peu d'espace dans le disque dur interne de l'appareil android. En effet, notre application a comme avantage que celle-ci occupe une faible portion en memoire et que celle-ci performante.

Application Compatible

Cette application android utilise une ancienne version du systeme d'exploitation android et donc est compatible avec un grand nombre d'appareil android.

Présentation de recherche


De nos jours, la technologie mobile prend de plus d’ampleur, alors la clientèle augmente conséquemment. Les téléphones intelligents sont utilisés pour toutes sortes d’utilisations, et lentement ils remplacent de plus en plus les ordinateurs. Les lecteurs mp3 sont passés de mode, et les utilisateurs de téléphones intelligents s’attendent à pouvoir écouter tous leurs titres sur leur appareil mobile. D’où l’utilité à se familiariser avec le développement mobile afin de pouvoir suivre les tendances des consommateurs.

Mis à part notre connaissance de Java, nous avons que très peu d’expérience avec le langage XML et le fonctionnement de Android Studio. Nous pouvons certainement nous fier sur nos connaissances acquises dans notre parcours éducatif pour nous éclairer sur le développement mobile, mais c’est majoritairement une expérience nouvelle pour nous.

Le sujet :

Le développement Android et l’utilisation de fichiers XML

L’enjeu :

Développer des connaissances qui nous seront très probablement utiles dans le monde du travail, puisque le développement mobile va en ampleur. Les fichiers XML sont très utiles comme stockage et nous pensions que ça ferait changement du SQL habituel.

L’expérimentation :

Nous avons consulté plusieurs sites Web susceptibles de pouvoir nous venir en aide dans le développement de notre application. Nous pensions avoir assez de ressources du début, mais pendant le développement nous avons réalisé plusieurs fois que nous avions à rechercher sur un nouveau sujet afin de continuer. Nous avons exclusivement utilisé le débogage de Android Studio, avec nos cellulaires LG G3 pour tester notre application. Souvent lors des crash de l’application lors du développement, il a été utile pour nous de consulter le logcat. Nous avons extensivement utilisé le débogage sur nos téléphones afin de mener notre application à terme, et aussi pour expérimenter au fur et à mesure la fonctionnalité de nos implémentations.

Les ressources : Cette rubrique sera plus extensivement approfondie dans la section de bibliographie, mais les principales ressources que nous avons utilisées ont été : Openclassrooms pour en apprendre sur le XML et sur le développement d’applications Android, stackoverflow pour nos nombreuses erreurs au cours du développement (particulièrement sur les adaptateurs de listes), mais aussi : Notre connaissance à priori du langage Java nous a beaucoup aidés, puisque certaines lignes de code étaient identiques. Nous avons également suivi quelques tutoriels YouTube pour nous familiariser avec le merge/commit de Android Studio. Nous nous sommes extensivement ressourcés durant notre développement applicatif.

Les limites :

Malencontreusement, nous n’avons pu utiliser le XML comme nous l’avions planifié. Durant notre développement nous avons appris à quel point Android est réglementé sur son utilisation de XML avec des balises personnalisées. Il nous a donné plusieurs problèmes jusqu’au point que nous avons trouvé une meilleure alternative : Stocker la librairie dans un Arraylist d’objets, ce qui nous a été assez utile. En quelque sorte, c’était mieux de se réorganiser de cette manière, puisque c’était beaucoup plus conforme à l’utilisation commune de Android (les XML avec des balises personnalisées sont très restreints dans Android Studio, et il voudra constamment afficher des erreurs pour restreindre leur utilisation) et aussi parce que ça nous a évité d’écrire du code personnalisé pour rien. Parfois c’est mieux de ne pas se briser la tête et se conformer aux méthodes communes beaucoup plus utilisées.

Acquisition de connaissances


Notre acquisition de connaissances dans le développement mobile était très satisfaisante, nous nous sommes familiarisés avec la création d’interfaces graphiques à l’aide de XML, l’interaction interactivités dans Android, les méthodes de base dans toute activité Android, comment avoir accès à des données dans la carte micro SD du cellulaire Android, comment utiliser Android Studio, comment construire des XML personnalisés (malgré qu’on ne s’en sois pas servi nous nous sommes tout de même approfondis avec notre cours sur openclassrooms alors nous en sommes assez satisfaits), comment manipuler des fichiers audios, comment paramétrer des recherches dans une carte SD Android etc.

Familiarisation avec la création d’interfaces graphiques sous XML

Dans les premières étapes de développement de notre application nous nous sommes familiarisés et éduqués sur comment produire des interfaces interactives avec un design esthétiquement plaisant. Par exemple, dans la liste nous avons appris à faire le conteneur d’entrées dans la liste ainsi que ses entrées, bref, de faire un XML qui contient un autre XML à l’intérieur, et de les faire interagir de manière désirée.

Comment utiliser Android Studio

Nous nous sommes familiarisés à ce logiciel basé sur Intelli-J, c’était une expérience d’apprentissage valeureuse, mais souvent on a trouvé le logiciel assez instable/lent. Nous avons été capables de l’exploiter à un potentiel satisfaisant avec notre projet.

Comment construire des XML personnalisés

Malgré qu’on ne s’en sois pas servi pour stocker notre librairie nous nous sommes tout de même approfondis avec notre cours sur openclassrooms. Nous avons même eu des bases en DTD. Nous avons inclus dans notre remise le XML construit qui allait initialement contenir notre librairie.

Interaction inter-activités dans Android

Nous avons acquis des connaissances sur comment passer d’une activité à l’autre (d’un écran à l’autre) et de même avoir la capacité d’envoyer de l’information d’une activité à une autre. Ceci nous a été très utile pour faire la transition entre la vue de la librairie (sur laquelle on vient de choisir une chanson à jouer) et la vue du lecteur de musique (qui recevra les informations de la chanson choisie et modifiera son interface et son comportement en accordance).


Comment manipuler des fichiers audio

Pour cette partie nous avons utilisé un outil assez facile d’emploi, un MediaPlayer. Malgré que son utilisation soit assez simple, elle était fondamentale à notre développement et nous sommes satisfaits de nous être familiarisés à lui, parce que c’est probablement très typique d’une application Android de jouer des fichiers audio ou vidéo.

Parcourir les données contenues dans la carte micro SD du cellulaire

Ceci a été particulièrement difficile à trouver et à personnaliser selon l’usage que nous voulions en faire. Nous avons appris l’existence des curseurs (des explorateurs de fichiers en quelque sorte) et du MediaStore, qui nous aide à obtenir des informations qui sont propriétaires aux fichiers audio, tels que le nom de la chanson, l’artiste et le nom de l’album. Ce code a été indispensable dans la fonctionnalité de notre application. De plus, il est extrêmement rapide.


Démarche expérimentale


Avant de commencer à développer notre application, nous nous sommes extensivement documentés sur les bases du développement Android, ceci a été une étape très importante parce qu’on devait se sentir à l’aise pour nous permettre de mener notre projet à terme.

L’apprentissage sur les feuilles XML dans Android nous a été primordial dès les premières étapes de développement, puisqu’habituellement c’est une des premières étapes sur lesquelles travailler lorsqu’on commence à développer une Activité.

D'approfondir sur les fonctionnalités du MediaPlayer, nous a été particulièrement utile. Nous avons plusieurs fois «débuggé» l’application en testant plusieurs fonctionnalités de base pour nous assurer que notre fichier audio était bel et bien joué et géré avec le MediaPlayer

Initialement, nous pensions faire une application qui effectue la lecture de fichier MP3 et qui effectue les opérations de base d’un MediaPlayer et je crois qu’au final l’application remplit bien nos critères et exigences espérées. Notre application est capable de détecter tous les fichiers audio sur la mémoire interne du système Android, elle est aussi capable de faire la sélection d’un titre et celle-ci possède plusieurs boutons permettant de mettre un titre sur pause, de changer de titre et de modifier la progression de ce titre.

Notre démarche expérimentale a été assez classique, au sens que nous déboguions l’application, jusqu’à temps à ce qu’on ait le résultat voulu. C’était majoritairement : de la documentation avec beaucoup d’essais erreurs, puisque nous n’avions pas vraiment de ressources qui nous montraient exactement ce que nous voulions accomplir. Ceci a été le cas avec les curseurs, avec les interfaces qu’on voulait implémenter et avec les contrôles du lecteur de musique. Lorsqu’une solution était ineffective, nous en cherchions une autre jusqu’à temps de trouver la bonne, ou encore mieux, nous avons combiné plusieurs solutions ensemble, notamment pour le cas des curseurs et de l’adaptateur personnalisé créateur de listes. De plus, à la fin de notre projet, nous avons extensivement testé notre interface pour nous assurer que tout était bel et bien fonctionnel.
